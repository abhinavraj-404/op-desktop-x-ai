Intelligence

It knows when it's lost.
And it recovers.

Two systems work in tandem — one detects failure in real time, the other learns from it over time.

stuck detection

Cyclic pattern guard.

After every action, the executor compares the new screen state against a rolling window of prior states. If the pixel diff falls below threshold for N consecutive steps — the agent is looping.

Rather than retrying blindly, it escalates to the Planner with a diagnostic snapshot: stuck_at_step=7, repeated_action=click(892,541), diff=0.002. The Planner replans from scratch with full context.

// stuck_detector.py
if consecutive_no_change >= threshold:
raise StuckException(
step=self.step,
last_action=action,
diff=pixel_diff
)
→ escalate to Planner
→ replan with diagnostic
learning from mistakes

Every failure is a lesson.

Task outcomes — success, failure, partial — are structured and stored in ChromaDB with vector embeddings. Each log captures the action sequence, error context, and final state diff.

The self-optimizer periodically queries these logs, surfaces recurring failure clusters, and proposes revised strategies. Over time, the agent builds a long-term memory of what doesn't work — and stops repeating it.

Capabilities

Built for production.

Multi-signal perception

Combines screenshots, macOS accessibility tree, OCR text extraction, and pixel-level screen diffing for full situational awareness.

Typed action schema

20 Pydantic-validated action types — click, type, scroll, drag, hotkey, open_app, spotlight_search, wait, replan — all schema-enforced before execution.

Task scheduling

Custom asyncio scheduler supports one-shot and recurring tasks with priority queues — daily reports, automated research, recurring workflows.

Skill library

Record, replay, and rate reusable action sequences. Teach once, reuse forever. Skills are keyword-searched and ranked by reliability score.

Vector memory

ChromaDB-backed semantic search over three collections: learned strategies, completed task outcomes, and general knowledge — injected into every LLM prompt.

macOS-native

Deep integration via PyObjC: ApplicationServices, Quartz, and Cocoa frameworks for native accessibility and screen capture.

Self-optimization

Analyses skill reliability and failure patterns from memory to surface prompt adjustment suggestions — implemented in learning/self_optimize.py.

Schema-enforced safety

Every action is Pydantic-validated before execution. No raw shell commands, no arbitrary code — typed, bounded, auditable.

Perception

Four signals. One picture.

Most agents see only screenshots. This one fuses four independent data streams before deciding what to do.

Screenshot
Full 1920×1080 RGBA capture via Quartz. Compressed and passed to the vision model at each step.
~180ms
Accessibility Tree
AXUIElement hierarchy via ApplicationServices. Extracts roles, labels, bounds, and enabled state for every visible node.
312 nodes avg
OCR
EasyOCR on regions where the accessibility tree returns empty labels. Catches canvas text, custom-rendered UI, and PDFs.
fallback layer
Screen Diff
Pixel-level structural similarity (SSIM) between consecutive frames. change_ratio drives stuck detection threshold.
SSIM · numpy
// perception/capture.py — fused signal per step
perception = {
"screenshot": capture_screen(), # Quartz CGWindowListCreateImage
"ax_tree": get_ax_tree(app), # AXUIElement hierarchy
"ocr_text": ocr_if_needed(screenshot), # EasyOCR fallback
"diff_ratio": ssim_diff(prev, screenshot), # change_ratio → stuck guard
}
→ passed verbatim to executor LLM context window

Log Schema

Every step is structured.

Tasks are logged as JSONL — one event per line. Machine-readable, auditable, and queryable by the self-optimizer.

Raw event — step_executed

{
  "event":      "step_executed",
  "timestamp":  "2026-04-20T13:23:30Z",
  "elapsed_s":  41.079,
  "step":       1,
  "action":     "open_app",
  "params":     { "app_name": "Safari" },
  "timing":     {
    "llm_decision_ms": 5085,
    "perception_ms":   653,
    "execution_ms":   2717,
    "total_step_ms": 8455
  },
  "verification": {
    "verified":      true,
    "screen_changed": true,
    "change_ratio":   null
  },
  "stuck_warning": null
}

Field reference

eventstring
task_started · plan_created · step_executed · task_completed · task_failed
timingobject
llm_decision_ms, perception_ms, execution_ms broken out per step for profiling
verificationobject
verified bool + screen_changed + change_ratio (SSIM). Drives stuck detection.
stuck_warningnull | object
Populated when consecutive no-change exceeds threshold. Triggers Planner escalation.
thoughtstring
Full LLM reasoning chain verbatim. Queryable by self-optimizer for failure analysis.
